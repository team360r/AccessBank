## Section 4: Dark Mode Done Right

Dark mode isn't just a visual preference — for many users with photosensitivity or migraine conditions, it's a necessity. But dark mode introduces its own contrast challenges. Colours that pass 4.5:1 on white backgrounds often fail on dark surfaces.

<div className="ab-step">
  <div className="ab-step__badge">1</div>
  <div className="ab-step__content">

Each theme variant needs its own contrast-tested palette. The `AppColors` class in AccessBank defines accessible colours for the light theme — but a dark theme needs different values for `onSurface`, `textSecondary`, and other roles.

  </div>
</div>

<Tabs>
  <TabItem value="before" label="Before — fails in dark mode" default>

```dart title="lib/theme/app_theme.dart"
// onSurface: #9E9E9E on dark surface #1E1E1E
// Contrast ratio: ~3.8:1 — FAILS WCAG AA for normal text (needs 4.5:1)
ThemeData darkTheme = ThemeData.dark().copyWith(
  colorScheme: const ColorScheme.dark(
    surface: Color(0xFF1E1E1E),
    onSurface: Color(0xFF9E9E9E), // ~3.8:1 — fails WCAG AA
  ),
);
```

  </TabItem>
  <TabItem value="after" label="After — passes in dark mode">

```dart title="lib/theme/app_theme.dart"
// onSurface: #D4D4D4 on dark surface #1E1E1E
// Contrast ratio: ~9.2:1 — passes WCAG AA AND AAA
ThemeData darkTheme = ThemeData.dark().copyWith(
  colorScheme: const ColorScheme.dark(
    surface: Color(0xFF1E1E1E),
    onSurface: Color(0xFFD4D4D4), // ~9.2:1 — passes WCAG AAA ✓
  ),
);
```

  </TabItem>
</Tabs>

<div className="ab-callout ab-callout--blue">
  <div className="ab-callout__header">💡 Material 3 shortcut</div>
  <p>If you use Material 3's dynamic colour system with a seed colour (<code>ColorScheme.fromSeed(seedColor: AppColors.primary)</code>), Flutter generates a full colour scheme with accessible contrast ratios built in — for both light and dark modes. It's the easiest way to get contrast right without manually testing every colour pair.</p>
</div>

## Section 5: Beyond Colour — Don't Rely on Colour Alone

About 300 million people are colour-blind — most commonly with red-green confusion (deuteranopia or protanopia). If your app uses colour as the *only* way to communicate information, those users miss it entirely.

<div className="ab-step">
  <div className="ab-step__badge">1</div>
  <div className="ab-step__content">

WCAG 1.4.1 ("Use of Colour") states that colour must not be the only visual means of conveying information, indicating an action, prompting a response, or distinguishing a visual element.

The fix: always pair colour with a secondary indicator — an icon, a text label, a shape, or a pattern.

  </div>
</div>

<div className="ab-step">
  <div className="ab-step__badge">2</div>
  <div className="ab-step__content">

In AccessBank, transaction amounts show green for positive and red for negative. A user with red-green colour-blindness sees both as the same brownish-grey. Adding directional icons (`arrow_upward` / `arrow_downward`) makes the distinction clear regardless of colour perception.

  </div>
</div>

<Tabs>
  <TabItem value="before" label="Before — colour only" default>

```dart title="lib/widgets/balance_indicator.dart"
// Colour alone — invisible to users with red-green colour blindness.
// Both "positive" and "negative" indicators look identical to ~8% of men.
Container(
  width: 8,
  height: 8,
  decoration: BoxDecoration(
    color: isPositive ? Colors.green : Colors.red,
    shape: BoxShape.circle,
  ),
)
```

  </TabItem>
  <TabItem value="after" label="After — colour + icon + semantic label">

```dart title="lib/widgets/balance_indicator.dart"
// Colour + directional icon + semantic label.
// The meaning is clear regardless of how you perceive colour.
Row(
  mainAxisSize: MainAxisSize.min,
  children: [
    Icon(
      isPositive ? Icons.arrow_upward : Icons.arrow_downward,
      color: isPositive ? AppColors.success : AppColors.error,
      size: 14,
      semanticLabel: isPositive ? 'Positive balance' : 'Overdrawn',
    ),
    const SizedBox(width: 4),
    Container(
      width: 8,
      height: 8,
      decoration: BoxDecoration(
        // Use accessible colour values with sufficient contrast
        color: isPositive ? AppColors.success : AppColors.error,
        shape: BoxShape.circle,
      ),
    ),
  ],
)
```

  </TabItem>
</Tabs>

<div className="ab-callout ab-callout--amber">
  <div className="ab-callout__header">Why this matters</div>
  <p>About 300 million people are colour-blind globally — roughly 8% of men and 0.5% of women. In a banking app, distinguishing positive from negative balances, successful from failed transactions, and healthy from overdrawn accounts is critical. Colour alone never communicates meaning reliably.</p>
</div>

Take a screenshot of AccessBank and run it through the red-green blind simulation. Can you still distinguish positive from negative amounts?</p>
</div>
