## Section 3: Spotting the Problems

Here are the five categories of accessibility failures present in AccessBank. These aren't edge cases — they are the top five issues found in real banking app audits.

<div className="ab-step">
  <div className="ab-step__badge">1</div>
  <div className="ab-step__content">

**Missing labels** — The quick-action icon buttons (transfer, pay, add, more) on the Account Overview have no text alternative. TalkBack and VoiceOver both announce "Button" — four times in a row. A user has absolutely no way of knowing what each button does.

  </div>
</div>

<div className="ab-step">
  <div className="ab-step__badge">2</div>
  <div className="ab-step__content">

**Meaningless labels** — The account card icon is announced as "Image". The balance is read as a raw number with no currency or context. These elements have *some* semantics, but the content is useless.

  </div>
</div>

<div className="ab-step">
  <div className="ab-step__badge">3</div>
  <div className="ab-step__content">

**Poor focus order** — The login screen's focus order doesn't match the visual layout. The screen reader jumps around unexpectedly, which is deeply disorienting when you're relying entirely on audio feedback.

  </div>
</div>

<div className="ab-step">
  <div className="ab-step__badge">4</div>
  <div className="ab-step__content">

**Low contrast** — The account name and balance text uses `AppColors.inaccessible.lightBlueOnWhite` — a colour with a contrast ratio of roughly **2.0:1** against the white card background. WCAG requires 4.5:1 for normal text. One in three people over 65 have vision impairment; this text is nearly invisible to them.

  </div>
</div>

<div className="ab-step">
  <div className="ab-step__badge">5</div>
  <div className="ab-step__content">

**Tiny touch targets** — The quick-action icon buttons are set to 32×32px with `EdgeInsets.zero` padding and `BoxConstraints(minWidth: 32, minHeight: 32)`. The WCAG recommended minimum is 44×44px. Users with motor difficulties (including many older adults) routinely miss small targets.

  </div>
</div>

<div className="ab-callout ab-callout--amber">
  <div className="ab-callout__header">Why this matters</div>
  <p>Every one of these issues blocks real users from banking independently. Fixing them isn't just compliance — it's giving people back their financial autonomy. And in many jurisdictions (UK, EU, Australia, Canada), accessible digital services are also a legal requirement.</p>
</div>

## Section 4: Understanding the Semantics Tree

Flutter renders your UI to a canvas — pixels, not DOM elements. Unlike the web, there's no HTML to inspect. So how does assistive technology know what's on screen?

<div className="ab-step">
  <div className="ab-step__badge">1</div>
  <div className="ab-step__content">

Flutter maintains a **parallel semantics tree** alongside the widget tree. This is a structured description of every meaningful element on screen — labels, roles, states, and available actions. When you add a `Semantics` widget (or use a built-in widget like `ElevatedButton` that already has semantics), Flutter updates this tree.

The OS accessibility service (VoiceOver or TalkBack) reads the semantics tree, not the pixel canvas. It's the bridge between your Flutter app and your users' assistive technology.

  </div>
</div>

<div className="ab-step">
  <div className="ab-step__badge">2</div>
  <div className="ab-step__content">

Here's the difference between a card with no semantics and one with proper labels. This is the core transformation you'll be making throughout this tutorial.

  </div>
</div>

```dart title="lib/screens/account_overview/widgets/account_card.dart"
// TalkBack announces: "Image" ... "4285.50"
// Two disconnected, meaningless elements. No context whatsoever.
Widget _buildInaccessibleVersion(BuildContext context) {
  final inaccessible = AppColors.inaccessible;

  return Card(
    margin: const EdgeInsets.symmetric(horizontal: 16, vertical: 6),
    child: Padding(
      padding: const EdgeInsets.all(16),
      child: Row(
        children: [
          Icon(
            _iconForType(account.type),
            // Low-contrast icon colour (~2.0:1 on white)
            color: inaccessible.lightBlueOnWhite,
            size: 32,
          ),
          const SizedBox(width: 16),
          Expanded(
            child: Column(
              crossAxisAlignment: CrossAxisAlignment.start,
              children: [
                // Low-contrast, no semantic label
                Text(
                  account.name,
                  style: TextStyle(
                    color: inaccessible.lightBlueOnWhite,
                    fontWeight: FontWeight.w600,
                  ),
                ),
                const SizedBox(height: 4),
                // Raw number, no currency context for screen readers
                Text(
                  '\$${account.balance.toStringAsFixed(2)}',
                  style: TextStyle(
                    color: inaccessible.lightBlueOnWhite,
                    fontSize: 18,
                  ),
                ),
              ],
            ),
          ),
        ],
      ),
    ),
  );
}
```

  </TabItem>
  <TabItem value="after" label="After — accessible" default>

```dart title="lib/screens/account_overview/widgets/account_card.dart"
// TalkBack announces: "Everyday Checking, checking account,
// balance four thousand two hundred eighty-five dollars and fifty cents"
// One clear, complete, human-readable announcement.
Widget _buildAccessibleVersion(BuildContext context) {
  final spokenLabel =
      '${account.name}, ${_typeLabel(account.type)}, '
      'balance ${_spellOutAmount(account.balance)}';

  return MergeSemantics(
    child: Semantics(
      label: spokenLabel,
      child: Card(
        margin: const EdgeInsets.symmetric(horizontal: 16, vertical: 6),
        child: Padding(
          padding: const EdgeInsets.all(16),
          child: Row(
            children: [
              // Decorative icon excluded — the spoken label covers its meaning
              ExcludeSemantics(
                child: Icon(
                  _iconForType(account.type),
                  color: AppColors.primary, // 7.3:1 contrast ratio
                  size: 32,
                ),
              ),
              const SizedBox(width: 16),
              Expanded(
                child: Column(
                  crossAxisAlignment: CrossAxisAlignment.start,
                  children: [
                    Text(
                      account.name,
                      style: const TextStyle(
                        color: AppColors.textPrimary, // 18.1:1 on white
                        fontWeight: FontWeight.w600,
                      ),
                    ),
                    const SizedBox(height: 4),
                    Text(
                      '\$${account.balance.toStringAsFixed(2)}',
                      style: const TextStyle(
                        color: AppColors.textPrimary,
                        fontSize: 18,
                      ),
                    ),
                  ],
                ),
              ),
            ],
          ),
        ),
      ),
    ),
  );
}
```

## Section 5: Exploring Semantics Nodes

<div className="ab-step">
  <div className="ab-step__badge">1</div>
  <div className="ab-step__content">

Toggle on the **AccessGuide inspector** (the magnifying-glass icon in the tutorial panel) and look at the Account Overview screen. Each coloured box represents a `SemanticsNode` — exactly the data that gets sent to TalkBack or VoiceOver.

- **Green border** — the node has a label ✓
- **Red border** — an interactive element with no label ✗
- **Grey border** — non-interactive, no label (often fine for decorative elements)

</div>
</div>
